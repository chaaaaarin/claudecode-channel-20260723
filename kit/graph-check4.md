# 🎁 グラフ化 要否4条件診断チェックリスト

そのループ、本当にグラフ化が必要か。4つの質問で判定します。

**受け取り方**: 下のコードブロック右上のコピーボタンでコピーするか、このファイルをそのままダウンロードしてください。

**使い方**: 今使っているAIループ（作る→レビュー→直す、を繰り返す仕組み）を1つ思い浮かべながら、上から順にチェックしてください。

**ポイント**: 2つ以上YESなら着手する価値あり。1つ以下ならまだループのままで十分です（無理にグラフ化しない）。

```text
【グラフ化 要否4条件診断チェックリスト】

■ ステップ1：視点を分けられるか確認する
□ 今のループは、役割の違う複数の視点（校閲・レビュー・チェック観点など）を1つのAIに同時にやらせていないか
→ YES: 視点ごとに独立したAI呼び出しに分割できないか検討する
指示例: このループを、視点ごとに分けるとしたら何個に分かれる？

■ ステップ2：並列化の余地を確認する
□ 本当は同時に進められる作業を、1つずつ順番にやらせていないか
→ YES: 並列実行（ファンアウト）の対象になる
指示例: このうち、同時に進めても問題ない工程はどれ？

■ ステップ3：制御フローを書き出せるか確認する
□ 「指摘・結果の種類によって戻り先が変わる」ような分岐が、その場の判断だけで決まっていないか
→ YES: 分岐図（グラフ）に書き出す価値がある
指示例: このループで、指摘の種類ごとに戻り先を分けるとしたら？

■ ステップ4：完了の基準そのものが変わったか確認する
□ グラフ化しても「何をもって終わりとするか」の基準は以前と同じままか
→ YES（同じまま）: 表面だけ複雑にしている可能性がある。基準も一緒に見直す
→ NO（基準も変えた）: 本質的にパラダイムが変わった可能性が高い

■ 判定
□ 2つ以上がYES → グラフ化の価値あり。まず1つのループに3行だけ足してみる
□ 1つ以下 → 今はループのままで十分。無理に複雑にしない
指示例: このループ、グラフ化すべきか判定して
```
